# 자기소개서 — .NET 백엔드 (주제별)

## 성격 및 개발 철학

저는 기능 구현보다 구조 설계를 먼저 고민하는 개발자입니다. C#을 3년 이상 독학하며 언어 문법을 넘어 설계 철학까지 파고든 덕분에, 눈앞의 기능을 빠르게 끝내기보다 각 책임을 어느 계층에 두어야 하는지부터 정리하는 습관이 몸에 배었습니다. 디지털 트윈 팩토리에서 비즈니스 로직과 UI가 뒤엉킨 상태를 MVVM 도입으로 근본부터 재구성해 UI 상태 관련 버그를 사실상 제거한 것이 이 성향의 대표적 결과물입니다.

동시에 필요한 영역이라면 낯선 스택이라도 곧바로 시작합니다. 같은 프로젝트에서 백엔드 공백이 생겼을 때 ASP.NET Core·EF Core·PostgreSQL을 독학해 REST API 서버를 단독 구축했으며, 이 경험으로 설계 중심 성향이 순수 로직과 아키텍처의 영역인 백엔드와 특히 잘 맞는다는 확신을 얻었습니다.

---

## 일하는 방식

모르는 문제를 만나면 먼저 공식 문서와 원문 사양을 직접 조사해 해결을 시도하고, 동료에게 질문할 때는 시도 경로와 맥락을 정리한 뒤 가볍게 의견을 구합니다. 막히는 과정 자체에서 배우는 것이 있기 때문에, 중요한 문제일수록 우회하기보다 원리까지 파고들어 해결합니다.

AI 도구는 업무에 도입하되 답을 얻는 데서 멈추지 않고 왜 그 답이 맞는지 근거를 검증하는 방향으로 씁니다. VR 도장 시뮬레이션에서 ML 추론 대신 정적 수식을 채택한 것도 VR 90fps 프레임 예산이라는 제약 조건을 근거로 한 판단이었으며, 엑셀 수식의 소수점 정밀도 한계를 발견했을 때는 Python으로 raw data를 재분석해 6차 다항식 계수를 재산출하고 C#으로 자동 포팅하는 스크립트까지 만들었습니다.

우선순위는 중요도 기준으로 판단하되 마감은 반드시 지키며, 명시적 요구사항이 없는 상황에서는 고객 의도를 파악해 필요한 기술을 스스로 찾아 적용하는 방식으로 일했습니다. 백엔드를 처음 맡게 된 계기도 그런 상황에서 출발했습니다.

---

## 경력 요약

2023년 8월부터 2025년 9월까지 2년 2개월간 주식회사 메타뷰에서 Unity 풀스택 개발자로 재직하며, 2년 차부터 약 1년 1개월간 디지털 트윈 팩토리 프로젝트의 메인 개발 리드를 맡았습니다.

백엔드에서는 ASP.NET Core·EF Core·PostgreSQL 기반 REST API 서버를 요구사항 분석부터 ERD 설계, CRUD 구현, JWT 인증, RBAC 권한 관리, FBX 파일 업로드 시스템까지 단독 구축했습니다. 클라이언트에서는 MVVM 패턴 도입으로 계층을 분리해 UI 상태 버그를 제거하고, 커스텀 바인딩을 R3 리액티브 프레임워크로 전환했으며, FBX→glTF 변환 파이프라인을 Blender CLI 기반으로 구축해 50~90MB 대형 모델의 디시리얼라이즈 시간을 5분에서 30초로 단축했습니다.

병행 프로젝트인 VR 도장 시뮬레이션에서는 X축 6차·Z축 2차 다항식 기반 3D 보간 함수를 설계해 VR 90fps 실시간 연산을 구현했고, 사내 Nvidia A40 GPU 서버 구축과 Ollama·LangChain RAG 기반 사내 LLM 챗봇 운영까지 맡아 인프라 영역으로 확장했습니다. 팀 내 MVVM 교육과 신입사원 코드리뷰 프로그램(2회)을 주도했습니다.

---

## 앞으로의 방향

지금까지의 제 성장 궤적은 클라이언트에서 시작해 백엔드로 옮겨갔고, 이후 Linux·Docker·LLM 인프라까지 영역을 넓혀온 과정입니다. 설계 중시 성향이 백엔드 아키텍처 설계와 특히 잘 맞는다는 것을 실무에서 확인한 뒤, 진로의 축을 .NET 백엔드 전문 영역으로 수렴시키기로 결정했습니다.

앞으로는 단순히 언어를 다루는 개발자가 아니라, .NET 기반 시스템 전반을 설계하고 운영 단계까지 책임지는 엔지니어로 성장하겠습니다. 필요한 기술을 직접 찾아 익히는 태도는 지금까지의 방식이었고 앞으로도 변하지 않습니다.
